# Selenium Automation Testing Project Guide

## Project Quick Start

To run this project on any system:

### Step 1: Activate virtual environment

Windows:

```bash
venv\Scripts\activate
```

Example:
This loads all project packages inside the virtual environment.

---

### Step 2: Install required packages

```bash
pip install -r requirements.txt
```

Example:
This installs Selenium, pytest, report tools, logging support, and browser drivers.

---

### Step 3: Run tests

```bash
pytest -v
```

Example:
The framework starts browser automatically and executes all test files.

---

### Step 4: Check reports folder

After execution:

```plaintext
reports/
```

Example:
HTML report shows passed tests, failed tests, duration, and summary.

---

### Step 5: Verify screenshots folder

If any test fails:

```plaintext
screenshots/
```

Example:
Framework automatically captures failure evidence.

# Phase Details

## Phase 1: Login Automation

Browser learned how to perform login like a human tester.

Example:

* Open login page
* Enter username
* Enter password
* Click login

Assertion checks whether login really succeeded.

---

## Phase 2: Negative Testing

Wrong input was given intentionally.

Expected:
System should show error.

Framework verified:

* Error message displayed
* Login blocked correctly

Example:
Wrong password test

---

## Phase 3: Screenshot on Failure

If any test fails:

Framework automatically saves screenshot.

Example:
If login button not found → screenshot captured instantly

This matches real QA bug evidence practice.

---

## Phase 4: HTML Reporting

Terminal output disappears after run.

HTML report gives permanent summary.

* Failed tests

Acts like a test dashboard.
